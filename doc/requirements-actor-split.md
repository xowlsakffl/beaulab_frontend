# Beaulab 액터 분리 요구사항 정의서 (스태프 / 병원 / 뷰티 / 사용자)

## 목적
- 기존 `partner` 액터를 `hospital`, `beauty`로 분리한다.
- 각 액터가 독립 로그인, 세션, 권한, 기능을 갖도록 요구사항을 정의한다.
- 현재 스태프 웹 구현 기능을 기준선으로 전 액터 요구사항을 정리한다.

## 표준 컬럼
- 요구사항 ID
- 요구사항명
- 기능ID
- 기능명
- 상세 설명
- 요청데이터 (`*` = 필수)
- 응답데이터 (수신 데이터 전체)
